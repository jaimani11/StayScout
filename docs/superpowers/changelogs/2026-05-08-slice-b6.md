# Slice B6 — `/destinations/[slug]` SEO + Mobile Bottom-Sheet

**Tag:** `slice-b6`
**Plan:** [`docs/superpowers/plans/2026-05-08-slice-b6-destinations-and-mobile.md`](../plans/2026-05-08-slice-b6-destinations-and-mobile.md)

## Goal

Static destination pages drive long-tail search; the mobile bottom-sheet wraps the existing workspace UX in a tappable native-feeling shell. The destinations curation library already had the core data — B6 wires routes, sheet, Open Graph, JSON-LD, and sitemap.

## What shipped

### Curation extension (`src/lib/curation/destinations.ts`)
- `headline: string` — Fraunces-italic short fragment (≤10 words)
- `oneLiner: string` — single editorial sentence (≤30 words)
- 7 curated entries pass the existing taste-lint banned-word filter

### Destination components (`src/features/destinations/`)
- `DestinationHero` — full-bleed hero photo with vignette, slow scale-in mount choreography, Fraunces-italic headline
- `FeaturedStays` — grid of 3–6 stays from `STAYS_BY_DESTINATION[slug]`. Click → `/?prompt=…`
- `PlanTripCta` — "Plan your trip to {name}" → workspace with prompt seeded
- `DestinationCard` — used by the index

### Routes
- `/destinations` — index page; static
- `/destinations/[slug]` — single destination; `generateStaticParams` over the 7 curated slugs; `generateMetadata` with Open Graph + Twitter card; 404s on unknown slug
- `DestinationJsonLd` component — Schema.org `TouristDestination` JSON-LD inlined
- `/sitemap.xml` — Next.js sitemap; includes `/`, `/destinations`, every `/destinations/[slug]`. Excludes `/t/[slug]` (share slugs are unguessable, not for indexing)
- `/robots.txt` — disallows `/api/`, `/t/`; allows the marketing surface

### Mobile bottom-sheet (`src/features/workspace/`)
- `useMediaQuery` + `useIsMobile` — SSR-safe hook (renders `false` on server + first paint, flips post-mount)
- `MobileBottomSheet` — three discrete snap points (peek 140px, half 50vh, full 95vh). Drag-to-snap with rubber-banding past bounds; flick velocity ≥ 600 px/s skips a snap. Drag handle is also a tap target that cycles snaps. Backdrop dims the canvas only at `full`.
- `Workspace` — switches between desktop split layout and the mobile sheet shell based on `useIsMobile()`. Same children components (Canvas, ChatSidebar) — different shell.

### Tests
- `tests/destinations-page-data.test.ts` — for each curated destination verifies: voice lint passes, headline ≤10 words, oneLiner ≤30 words, slug resolves, mood snapshot exists, ≥3 featured stays, first stay has a usable photo. 7 destinations × 6 invariants = **+42 tests**.

## Architectural decisions (recorded)

**1. Destination pages are static (SSG).** 7 curated entries, hand-written copy — `generateStaticParams` + edge-cacheable. Adding a destination means a curation library change anyway.

**2. JSON-LD structured data, not just OG.** Schema.org `TouristDestination` lifts SERP rich results — small bytes cost, measurable upside.

**3. The CTA is `?prompt=…`, not a separate API.** B3's `UrlInit` already consumes `?prompt=` on first paint and strips it from the URL. Destination CTAs reuse that — zero new flow plumbing.

**4. The mobile bottom-sheet is desktop-invariant.** Desktop split layout untouched. Below `md` we swap the shell, not the children. Breakpoint logic lives in one place; `useIsMobile()` is SSR-safe so no hydration flash.

**5. Discrete snap points, not free-drag.** Three snaps mirror native iOS/Android sheet UX. Free-drag is a known UX trap (users get stuck at random heights).

**6. SEO content respects the editorial voice.** All 7 headline + oneLiner pairs pass `containsBannedWord`. Italic fragments, no "discover/journey/unforgettable."

**7. Mock-safe end-to-end.** Featured stays come from the curated mock-italy data; works with zero env keys. Real-provider augmentation via `searchWithFanout` (B5) is a B6.x polish, not a requirement.
