# Slice B2 - Orchestrator → LangGraph (mock-safe, opt-in rollout)

**Tag:** `slice-b2`
**Plan:** [`docs/superpowers/plans/2026-05-08-slice-b2-orchestrator-langgraph.md`](../plans/2026-05-08-slice-b2-orchestrator-langgraph.md)
**Spec:** [`docs/superpowers/specs/2026-05-08-stayscout-slice-b-design.md`](../specs/2026-05-08-stayscout-slice-b-design.md)

## Goal

Replace the hand-rolled imperative orchestrator with a LangGraph `StateGraph`, preserving the public `run()` AsyncIterable contract and event stream byte-for-byte. Two engines coexist behind a `STAYSCOUT_ORCHESTRATOR` flag during rollout; the hand-rolled engine remains the default until parity is proven against real traffic.

## What shipped

### LangGraph engine
- `src/orchestrator/langgraph/state.ts` - `Annotation.Root` channel definition (`request`, `priorTurn`, `intent`, `searchResult`, `proposal`, `proposalRef`, `agentTrace`, `hardEnded`, `softEnded`, `turnStartedAt`).
- `src/orchestrator/langgraph/nodes.ts` - seven node factories (`bootstrap`, `extract_intent`, `run_search`, `compose_proposal`, `run_mood`, `evaluate_hint`, `complete_turn`) plus three routers. Each node closes over `GraphDeps` (modelClient, agents, providerRouter, sessionStore, per-instance hinter/seenSessions/seenTurnIds Maps).
- `src/orchestrator/langgraph/graph.ts` - `buildGraph(deps, checkpointer?)` wires the StateGraph: `START → bootstrap → intent → search → (compose | complete | END) → mood → memory_hint → complete → END`.
- `src/orchestrator/langgraph/event-queue.ts` - typed AsyncIterable-backed queue. Events emit via a side-channel (`RuntimeContext.emit` plumbed through `RunnableConfig.configurable`), not through graph state. Time-ordered; the queue closes when the graph invocation resolves.
- `src/orchestrator/langgraph/orchestrator.ts` - `LangGraphOrchestrator` class. Same `run(req, ctx) → AsyncIterable<OrchestratorEvent>` shape as the legacy `Orchestrator`. Holds the same per-instance Maps (sessions, turns, hinters) so refine flow and idempotency work across turns within a process.

### Checkpointer (mock-safe)
- `src/orchestrator/langgraph/checkpointer.ts` - `getCheckpointer()`:
  - `MemorySaver` (LangGraph built-in) when `DATABASE_URL` is unset. Process-local; refine state survives across turns within a Node process.
  - `PostgresSaver` from `@langchain/langgraph-checkpoint-postgres` when `DATABASE_URL` is set. Tables live in the `langgraph` schema (separate from Prisma's `public`) so migrations don't conflict. `setup()` runs lazily on first construction; failure falls back to MemorySaver with a console warning.

### Engine factory + feature flag
- `src/orchestrator/engine.ts` - `createOrchestratorEngine()` (async). Reads `STAYSCOUT_ORCHESTRATOR` (default `hand-rolled`, opt-in `langgraph`). Constructs the right engine with shared deps. Both engines satisfy the structural `OrchestratorEngine` interface.
- `src/orchestrator/singleton.ts` - `getOrchestrator()` is now `async`. Caches the constructed instance + the kind it was constructed with. Hot-swap is intentionally not supported - flipping the flag mid-process would corrupt in-flight refine state.
- `src/lib/env/get-server-features.ts` - adds `orchestratorEngine` to the surface.
- `src/app/api/concierge/route.ts` - `await getOrchestrator()` (one extra await; otherwise unchanged).

### Tests
- `tests/langgraph-orchestrator.test.ts` - mirrors `tests/orchestrator.test.ts` against `LangGraphOrchestrator`. 7 tests: compose sequence, mood ordering, session de-dup, refine, duplicate-turn guard, intent failure, AbortError classification.
- `tests/orchestrator-parity.test.ts` - runs identical `ConciergeRequest`s through both engines, normalizes per-run fields (timestamps, durations, stepIds, `freshness.fetchedAt`, `proposal.generatedAt`), and asserts deep-equal event streams. 3 scenarios: compose, refine, duplicate-turn.

## Architectural decisions (recorded for B3+)

These shaped the implementation; departures need explicit justification.

**1. Two engines coexist during rollout.** The hand-rolled engine is the default. `STAYSCOUT_ORCHESTRATOR=langgraph` opts in. Removal of the legacy engine is a future sub-slice - gated on parity holding for ≥2 weeks against real traffic and Postgres-checkpointer recovery being verified.

**2. Public API is invariant.** `run()` returning `AsyncIterable<OrchestratorEvent>` is the contract. The Next route handler, the singleton, all 16 orchestrator-related tests - none change.

**3. Minimal graph state.** Channels carry only what's needed for branching + checkpointing. Events emit via a side-channel (queue), not state. Reason: smaller state = smaller checkpoints = faster persist; events flow time-ordered, not batched at node end.

**4. Streaming via emitter, not LangGraph stream-mode.** Each node calls `runtime.emit(event)` from the `RunnableConfig.configurable` bag. We do not use LangGraph's `streamMode='updates'` - its event shape doesn't map cleanly to our discriminated union, and node-end batching loses within-node ordering (e.g., `step.completed` then `intent.extracted`).

**5. Stateful concerns isolated per Orchestrator instance.** `seenSessions`, `seenTurnIds`, `hinterBySession` live on the `LangGraphOrchestrator` instance - not in graph state. These are runtime invariants that must survive across turns within the same process; they're not part of the graph's data flow.

**6. Two termination modes.** `hardEnded` skips downstream work AND the `complete_turn` node (no `turn.completed`). `softEnded` skips compose/mood/memory but still runs `complete_turn` (empty-search path). This preserves legacy behavior exactly.

## Mock-safe matrix

| Var | Off (default) | On |
|---|---|---|
| `STAYSCOUT_ORCHESTRATOR` | hand-rolled engine | langgraph engine |
| `DATABASE_URL` (with langgraph) | MemorySaver - process-local checkpoints | PostgresSaver - durable, cross-restart |

The default path is unchanged: no flag, no DB, hand-rolled engine, in-memory state. `STAYSCOUT_ORCHESTRATOR=langgraph` flips the engine without requiring a DB. Adding `DATABASE_URL` upgrades the LangGraph checkpointer.
