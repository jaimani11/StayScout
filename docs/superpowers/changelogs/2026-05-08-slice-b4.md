# Slice B4 - Affiliate Redirect + Click Attribution

**Tag:** `slice-b4`
**Plan:** [`docs/superpowers/plans/2026-05-08-slice-b4-affiliate-redirect.md`](../plans/2026-05-08-slice-b4-affiliate-redirect.md)

## Goal

Replace the Slice A "ships in B" placeholder with a real server-side redirect + click attribution. The detail panel's "Continue to Booking" CTA now hits `/api/go`, which records an `AffiliateClick` row and 302s to the provider's deep link. Conversion reconciliation in B7+ lands on top of the rows this slice writes.

## What shipped

### SessionStore extensions
- `recordClick({stayId, providerId, affiliateUrl, ownerKind, ownerId, sessionId, turnId?, conversationId?})` returns the recorded row. Both `InMemorySessionStore` (process-local, `_getClicks()` test helper) and `PostgresSessionStore` (Prisma `affiliateClick.create` + `ensureUser` for authenticated owners) implement.
- `AffiliateClickRecord` shape - `userId` is null for anonymous; `sessionId` is always recorded so cross-device authenticated reconciliation has the anchor.
- 2 new contract tests in `tests/session-store.test.ts`.

### Allowlist + URL helper (`src/lib/affiliate/`)
- `AFFILIATE_HOST_ALLOWLIST` - exact-match or subdomain match on `example.com`, `booking.com`, `expedia.com`, `hotels.com`, `vrbo.com`, `airbnb.com`, `hotelbeds.com`, `skyscanner.com`, `viator.com`, `getyourguide.com`. New providers in B5 extend this list.
- `isAllowedAffiliateHost(url)` - protocol must be `https:`, host must exact-match or end with `.<allowed>`. Rejects `http`, malformed URLs, `javascript:`, naive substring matches like `fakebooking.com` or `expedia.com.attacker.io`.
- `generateGoUrl({stay, turnId?, conversationId?})` - relative `/api/go?s=...&p=...&u=...&t=...&c=...` with short keys. Omits empty optional params.
- 6 allowlist tests + 4 go-url tests.

### Redirect route
- `GET /api/go` - parses `s`/`p`/`u`/`t`/`c`, validates `u` against the allowlist, resolves owner from `getServerAuth()` + `ownerOf()`, calls `recordClick`. `recordClick` is wrapped in try/catch - DB failures log but do NOT block the redirect (booking flow is sacred). 302s to `u`.
- 400 on missing required params or allowlist fail. No 5xx visible to the user - even if recordClick throws, they get to the provider.

### Detail panel
- `confirm-redirect-modal.tsx` rewritten - drops the "Slice A demo / ships in Slice B" placeholder. Header reads "Continuing to {host}" derived from `stay.bookingLink.url`. CTA is a real `<a href={generateGoUrl(stay, {turnId})}>` with `target="_blank"` so the user keeps StayScout in the original tab.
- `detail-panel.tsx` passes `turnId={turn.turnId}` to the modal so the click row attributes back to the turn that produced the stay.
- Voice unchanged - italic Fraunces, no exclamations, no "discover/journey/unforgettable."

## Architectural decisions (recorded)

**1. Server-side 302, not client-side navigation.** Affiliate networks set first-party cookies on the redirect hop that survive cross-origin to the provider; client-side `location.href = ...` doesn't get those cookies.

**2. Open-redirect prevention is mandatory.** Without the allowlist, `/api/go?u=https://evil.example.com` becomes a phishing aid (StayScout's domain in the bar, attacker's site rendering). Allowlist-driven validation is the standard defense.

**3. Click is recorded before redirect, never blocking.** A failed DB write must NOT block the redirect - booking flow is sacred. `recordClick` is wrapped in try/catch + console.error.

**4. Owner derived from auth state.** `userId` for authenticated, `sessionId` for anonymous. `sessionId` is always recorded (even for authenticated owners) so cross-device reconciliation has an anchor.

**5. Mock-safe end-to-end.** Both stores implement `recordClick`. In-memory clicks survive within a dev session; Postgres clicks are durable.

**6. The redirect URL stays clean.** Short keys (`s`, `p`, `u`, `t`, `c`), all ASCII, encoded once. The 302 destination is `u` exactly as the provider supplied it.

**7. Click attribution doesn't authenticate.** Anonymous users click too. Spoofing `sessionId` only attributes a click to a fake bucket - there's no privilege gain.
