# Slice B1 - Persistence + Auth (mock-safe)

**Tag:** `slice-b1`
**Plan:** [`docs/superpowers/plans/2026-05-08-slice-b1-persistence-and-auth.md`](../plans/2026-05-08-slice-b1-persistence-and-auth.md)
**Spec:** [`docs/superpowers/specs/2026-05-08-stayscout-slice-b-design.md`](../specs/2026-05-08-stayscout-slice-b-design.md)

## Goal

Make the app durable and user-aware without breaking the keyless dev experience. Saved trips persist; signing in promotes the anonymous bucket onto the new user.

## What shipped

### Persistence boundary
- `SessionStore` interface in [`src/lib/session/session-store.ts`](../../../src/lib/session/session-store.ts) is the single seam between orchestrator/routes and storage. Owns turns, saved trips, and migration.
- Two implementations behind one shape:
  - `InMemorySessionStore` - process-local Maps. Default.
  - `PostgresSessionStore` - Prisma-backed. Active when `DATABASE_URL` is set and not the placeholder.
- Factory in [`factory.ts`](../../../src/lib/session/factory.ts) picks at call time.

### Schema
- [`prisma/schema.prisma`](../../../prisma/schema.prisma): `User`, `Conversation`, `Turn`, `Trip`, `AffiliateClick`, `MemoryRecord`. No Postgres-specifics - pgvector + array types arrive in B3/C1.
- `User.id` is the canonical owner key for both Clerk userIds AND `anon_<uuid>` session ids.
- `User.migratedFrom` audits anonymous → user moves so re-runs no-op.
- `@@unique([userId, proposalId])` on `Trip` makes save idempotent.

### Auth abstraction
- [`AuthState`](../../../src/lib/auth/auth-state.ts) discriminated union; `ownerOf()` resolves to `{ ownerKind, ownerId }` for storage queries.
- `AuthProvider` chooses between `AnonymousAuthProvider` and `ClerkAuthProvider` at render time based on `clientFeatures.auth` (build-time constant from `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`).
- `ClerkAuthBridge` derives `AuthState` from `useUser()` at render time - no setState in effects.
- Server-side `getServerAuth()` reads cookies + calls Clerk's `auth()` with try/catch fallback to anonymous.

### Conditional middleware
- [`src/middleware.ts`](../../../src/middleware.ts) does two things:
  1. Mints anonymous session cookie on first request (so `cookies()` in layout sees it).
  2. Delegates to Clerk's middleware only when auth is configured.
- Dynamic import keeps Clerk completely off the keyless build path.

### Trip routes
- `POST /api/trips/save` - bookmark a proposal for the current owner.
- `GET /api/trips/list` - list saved trips for the current owner.
- `GET /api/trips/[tripId]` - fetch one (404s if not owned).
- `DELETE /api/trips/[tripId]` - unbookmark (404s if not owned).
- `POST /api/auth/migrate` - promote anonymous bucket to user-owned. Idempotent.

### UI
- `SavedTripsPanel` slide-in panel, header `Saved` button with count badge.
- `Save this trip` CTA in detail panel - saves the proposal containing the currently-viewed stay.
- `MigrateOnSignIn` effect fires `/api/auth/migrate` on the anon → auth transition.

## Mock-safe invariant

Every external system is opt-in:

| Flag | Off (default) | On |
|---|---|---|
| `DATABASE_URL` | InMemorySessionStore | Prisma + Postgres |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` + `CLERK_SECRET_KEY` | Anonymous only, all session-owned | Clerk sign-in + auto-migration |
| `ANTHROPIC_API_KEY` | Mock IntentAgent + mood fixtures | Live Claude |

Mixing is supported. Set just `DATABASE_URL` to persist anonymous trips. Set just Clerk to sign in without persistence. The app runs end-to-end with all of them empty.
