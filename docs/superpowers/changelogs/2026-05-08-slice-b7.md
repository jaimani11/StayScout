# Slice B7 - Langfuse Traces + Cost/Latency Dashboard

**Tag:** `slice-b7`
**Plan:** [`docs/superpowers/plans/2026-05-08-slice-b7-langfuse-and-dashboard.md`](../plans/2026-05-08-slice-b7-langfuse-and-dashboard.md)

## Goal

Wire the existing `TraceLogger` seam to a real telemetry sink (Langfuse) and a local-first cost/latency dashboard at `/admin`. Per-turn cost + per-agent latency surface for the operator without external dependencies.

## What shipped

### Cost computation (`src/lib/observability/costs.ts`)
- `MODEL_PRICING` table - Claude 4.5/4.6/4.7 in Haiku/Sonnet/Opus tiers, USD per 1M tokens
- `computeCostUsd(model, tokensIn, tokensOut)` - returns USD or `null` for unknown models
- Unknown-model warnings logged once per model, not per-call
- 6 tests including pricing consistency invariants

### Memory telemetry store (`src/lib/observability/memory-telemetry-store.ts`)
- Process-singleton ring buffer: 200 turns × 10 agent runs each, FIFO eviction
- `beginTurn` / `recordAgentRun` / `completeTurn` / `failTurn` lifecycle methods
- `getSummary()` aggregates: turns/completed/failed counts, total cost, P50/P95 turn duration, per-agent P50/P95 latency
- HMR-safe via `globalThis` cache so dev reloads don't reset
- 8 tests covering eviction caps, summary aggregates, percentile math

### TraceLogger sinks
- `MemoryTraceLogger` - writes lifecycle + agent runs to the buffer; computes cost at sink time from `modelMeta`
- `LangfuseTraceLogger` - dynamic-imports `langfuse` SDK only when keys present. One trace per turnId, one generation/observation per agent run. SDK errors caught at every callsite - telemetry NEVER blocks the request flow
- `CompositeTraceLogger` - stacks N sinks; per-sink try/catch isolates failures (3 tests)
- `getTraceLogger()` factory - composes Memory + (Langfuse if keys) and caches per-process

### Admin dashboard (`/admin`)
- Server-rendered, `dynamic = 'force-dynamic'` so it always reflects the live buffer
- Auth gate: when Clerk is on AND `STAYSCOUT_ADMIN_PUBLIC` is unset, `/admin` redirects unauthenticated users to `/`. Without Clerk it's open (keyless dev convenience). `STAYSCOUT_ADMIN_PUBLIC=1` overrides for staging/preview deploys.
- 4 summary cards: Turns / P50 latency / Total cost / Error rate
- Agent latency chart - CSS-only horizontal bars showing P50 (accent) vs P95 (emphasis) per agent, sorted by P95 desc
- Recent turns table - turnId tail, type, sessionId tail, duration, agent count, cost. Status dot (green/red/grey) per row
- Empty states for "no turns yet" and "no agent runs yet"

### Engine wiring
- `engine.ts` - `createOrchestratorEngine()` now wires `getTraceLogger()` and wraps the inner orchestrator's `run()` so every yielded event also flows through `traceLogger.recordEvent`. Both legacy and LangGraph engines get telemetry without per-engine plumbing.

## Architectural decisions (recorded for B-late + C)

**1. The TraceLogger interface is invariant.** Adding cost/latency means new metadata in `modelMeta`, not new methods. Every existing agent caller is unchanged.

**2. Telemetry sinks are stackable.** `CompositeTraceLogger` composes Memory + Langfuse + future sinks (OpenTelemetry, etc.) under one interface.

**3. Cost is computed at sink time.** Agents provide tokens; sinks consult the price table. Pricing changes don't ripple into agent code.

**4. Mock-safe: no Langfuse keys = no Langfuse import.** Dynamic-import inside `LangfuseTraceLogger.init()` keeps the keyless build path off the SDK runtime.

**5. The in-memory buffer is bounded.** 200 turns × 10 runs each. FIFO eviction. No persistence - Langfuse handles archival; this is the operator's "what just happened?" view.

**6. The dashboard is auth-aware but not auth-required.** Three modes: keyless dev (open), Clerk on (auth required), `STAYSCOUT_ADMIN_PUBLIC=1` (open in any mode for staging/preview).

**7. Failure isolation everywhere.** Every Langfuse SDK call wrapped in try/catch. Composite sink isolates per-sink failures. Engine wrapping catches `recordEvent` failures. Telemetry NEVER blocks the user-visible path.
