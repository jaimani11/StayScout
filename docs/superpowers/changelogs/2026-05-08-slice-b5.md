# Slice B5 - Real Provider Integrations

**Tag:** `slice-b5`
**Plan:** [`docs/superpowers/plans/2026-05-08-slice-b5-real-providers.md`](../plans/2026-05-08-slice-b5-real-providers.md)

## Goal

Make the system ready to plug in real provider APIs (Booking.com, Expedia, Vrbo, Hotelbeds) without breaking the keyless dev experience. Ship a reference real-provider implementation pattern, an availability-aware registry, response caching, and multi-provider fanout - all mock-safe.

## What shipped

### Shared infrastructure (`src/providers/_shared/`)
- `cache.ts` - `LRUCache<K, V>` with per-entry TTL + insertion-order eviction. 8 tests covering eviction, TTL, LRU bumping, no-op TTL.
- `canonical-query.ts` - `canonicalizeQuery(q)` produces stable cache keys: order-independent for arrays, drops `compareSet` (ranking input, not search universe), JSON-safe. 5 tests.
- `http.ts` - `httpJson<T>(url, opts)` fetch wrapper with internal timeout + AbortSignal composition + 1 retry on 5xx. Throws `ProviderError` subclasses; returns null on 404. Strips query string from logs (affiliate ids + api keys live there).
- `affiliate-provider.ts` - `BaseAffiliateProvider` abstract class. Subclasses implement `fetchStays` + `buildBadges`; the base handles cache lookup → fetch on miss → set + return with `freshness: 'live' | 'cached'`.

### Booking.com reference provider (`src/providers/booking-com/`)
- `types.ts` - Zod schema for the Booking.com Demand API response (subset: `hotel_id`, `hotel_name`, location, photos, pricing, capacity, accommodation type).
- `client.ts` - `searchBookingCom(query, creds, signal)` constructs the request URL with affiliate id + Bearer auth, sends via `httpJson`, validates with Zod. Soft-fail on validation errors (returns empty result + logs) so the fanout falls through to other providers.
- `mapper.ts` - `mapBookingHotel(raw, affiliateId)` → `Stay`. Maps `accommodation_type_name` to our enum (Villa, Apartment, B&B → guesthouse, etc.). Builds the affiliate URL with `aid` + `hotel_id` + `label=stayscout` baked in. 6 mapper tests.
- `index.ts` - `BookingComProvider` class extending `BaseAffiliateProvider`. `fromEnv()` returns `null` when keys are missing - registry skips it.

### Availability-aware registry (`src/providers/index.ts`)
- `buildProviderRegistry()` - checks env, instantiates available real providers, returns `{all, real, mocks}`. Cached on first build; `_resetProviderRegistryForTesting()` invalidates for env-mocking tests.
- `routeProvider(intent)` - back-compat single-provider router. Returns the most-specific available provider: real provider that covers the destination > MockItaly for Italy queries > LLMSynthesized fallback.
- `routeProviders(intent)` - fanout list. Returns every provider that can serve the destination.
- `searchWithFanout(providers, query, ctx)` - `Promise.allSettled` parallel search + dedupe by stay id. One provider's outage doesn't stall the others. `freshness.source` is `'cached'` only if every contributing provider returned cached; otherwise `'live'`.
- `listAvailableRealProviders()` - diagnostic surface for `getServerFeatures`.
- 9 registry/fanout tests.

### Env + features surface
- `.env.example` - new `BOOKING_COM_AFFILIATE_ID` + `BOOKING_COM_API_KEY` block with note that adding both turns the provider on. Commented stubs for Expedia.
- `getServerFeatures()` - adds `providers: { bookingCom: boolean }`. Extensible - new providers slot in here.
- 2 features tests for the new field.

## Architectural decisions (recorded for B-late + C)

**1. Availability is per-instance, not per-class.** A `BookingComProvider` instance is constructed only when `BOOKING_COM_AFFILIATE_ID` + `BOOKING_COM_API_KEY` are both set. Without them, the class isn't instantiated - the registry doesn't expose it.

**2. Real providers always have HTTP + mapper + cache.** The base class handles all three. New providers are tiny: declare endpoint + auth, implement two methods.

**3. Cache is aggressive but bounded.** 30-minute TTL on availability, in-memory LRU max 500 entries per provider. Misses fall through; hits skip the API call.

**4. Router fanout is parallel + isolated.** `Promise.allSettled` so one provider's failure doesn't stall the others.

**5. Capabilities drive routing.** Each provider declares `regions: string[]` (or undefined for global). Router checks intent destination country.

**6. Mock-safe is the invariant.** Without ANY real-provider keys, behavior is identical to Slice B4. Setting one key flips one provider on; setting all keys flips all on.

**7. Affiliate URLs go through the existing /api/go redirect.** B4's allowlist already covers booking.com; new providers add a hostname.

## Out of B5 scope (deferred)

- Real Expedia, Vrbo, Hotelbeds, Skyscanner, Viator providers. Pattern is reusable; concrete impls land per-partner-approval as keys arrive.
- Per-provider rate-limit budget. Cache + provider-side rate limits cover v1.
- Distributed cache (Redis). In-memory works for one-pod dev; revisit at scale.
- Provider health metrics + circuit breaker. Fanout's `Promise.allSettled` already isolates failures.
