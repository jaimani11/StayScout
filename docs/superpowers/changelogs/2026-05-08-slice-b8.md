## Goal

Pay down four "out of scope" items from B1–B7 that turn into rough edges if left for Slice C. Pick high user-visible payoff with small architectural surface.

## What shipped

### Task 1 - `ModelClient.generateWithMeta` + IntentAgent reports cost (B7.x)
- `ModelClient` interface widened with `generateWithMeta<T>(req): Promise<{ result; modelMeta }>`. `generate` stays as a convenience wrapper - every existing call site is unchanged.
- `AnthropicModelClient` extracts a private `generateInternal` that returns both result + modelMeta from the Anthropic response object's `usage` field. Exposes `cacheHit` when `cache_read_input_tokens > 0`.
- `MockModelClient` (test helper) implements `generateWithMeta` with synthetic tokens.
- `IntentAgent.run` calls `generateWithMeta` and passes `modelMeta` to `recordAgentRun` - every intent extraction now surfaces cost on `/admin`.

### Task 2 - Refining a resurfaced saved trip works (B3.x)
- `POST /api/trips/[tripId]/resurface` - owner-gated, idempotent. Loads the trip, calls `sessionStore.putTurn` with a synthetic `TurnRecord` (turnId = `trip.id`).
- `useSavedTrips.resurface(tripId)` - fire-and-forget POST. Errors logged but never thrown.
- `saved-trips-panel.handleSelect` calls `resurface()` BEFORE pushing local state. Network never gates UX (resurface still happens locally even if the POST fails).
- Result: clicking a saved trip then refining it now works end-to-end. Previously the orchestrator's `getTurn(priorProposalRef.turnId)` would return null and the refine path silently degraded.

### Task 3 - Circuit breaker on `BaseAffiliateProvider` (B5.x)
- `circuit-breaker.ts` - three-state breaker (closed / open / half-open) with configurable threshold (default 5) + cooldown (default 30s). Standard transitions: closed → open after threshold, open → half-open after cooldown, half-open → closed on success or → open on failure. Per-instance, not global.
- `BaseAffiliateProvider` wraps `fetchStays` with `breaker.run(...)`. When the breaker rejects fast (or fetchStays throws), the provider returns an empty result instead of bubbling the error - fanout falls through to other providers automatically. Errors logged at warn so operators can see them.
- 7 tests covering all transitions + the consecutive-failure counter reset on success.

### Task 4 - Expedia reference provider (B5.y)
- Mirrors `booking-com/` file-for-file: `types.ts` (Zod schema for EPS Rapid response subset), `client.ts` (HTTP Basic auth from apiKey:sharedSecret), `mapper.ts` (Expedia property → Stay), `index.ts` (`ExpediaProvider extends BaseAffiliateProvider`).
- Self-registers via `fromEnv()` when `EXPEDIA_API_KEY` + `EXPEDIA_SHARED_SECRET` are both set.
- 6 mapper tests + 2 registry tests verifying both Booking.com + Expedia register together when their respective keys are set.
- `getServerFeatures().providers.expedia` flag.
- `.env.example` updated.

## Architectural decisions (recorded)

**1. The `Agent` contract stays invariant.** `generateWithMeta` is additive; `generate` is unchanged. Existing call sites keep working without modification.

**2. Resurface server-priming is opportunistic, not authoritative.** If the POST fails, the local resurface still proceeds; refine just won't work for that turn until network catches up. Network never blocks UX.

**3. Circuit breaker per-provider, not global.** A failing Booking.com mustn't suppress Expedia. The breaker is an instance field on `BaseAffiliateProvider`.

**4. Expedia mirrors Booking.com structurally.** Same files, same shape, same patterns. Anything that *needed* to change to add a second provider would expose a leak in the B5 base; the goal was for nothing to change. (Nothing changed - base class held.)

**5. Mock-safe is still the invariant.** Without keys, Expedia is invisible. Without `LANGFUSE_*`, IntentAgent's modelMeta still records to the in-memory buffer (cost on `/admin` for free).
