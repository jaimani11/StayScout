## Goal

Persistent semantic memory across sessions + devices. The IntentAgent's prompt is enriched with relevant prior preferences ("we always travel in September", "family of 4, walkable, slow"); the workspace surfaces a memory hint when retrieval contributed. Mock-safe end-to-end — the `pnpm dev` demo gets useful retrieval on the second turn without keys or DB.

## What shipped

### Embedding + retrieval primitives (`src/lib/memory/`)
- `BagOfWordsEmbedding` — deterministic local embedding (256-dim sparse, L2-normalized, djb2 token hashing). Always available, zero cost.
- `cosineSimilarity` — bounded in [-1, 1].
- `MemoryStore` interface + `InMemoryMemoryStore` — process-local ring buffer, owner-isolated, exhaustive cosine search with score floor + top-K.
- `MemoryRecorder` — observes completed turns, persists up to 3 records (episodic rawInput, structural intent snapshot, optional caveats). Idempotent on `turnId`. Failures caught + logged — never block the user.
- `MemoryRetriever` — wraps `MemoryStore.search` with score floor + prompt-block formatter. Returns `null` when nothing meets the floor.
- `getMemorySubsystem()` factory — process-singleton; cached `{ store, recorder, retriever, embedding, kind, embeddingKind }`.

### Schema
- `MemoryRecord.embedding Unsupported("vector(256)")?` field added to Prisma schema. Populated when `STAYSCOUT_PGVECTOR=1` (full migration + impl land in C1.x).

### IntentAgent + orchestrator wiring
- `IntentAgentInput.priorMemoryBlock?` — pre-formatted `<memory>` block from retrieval. Agent appends to the user message (NOT system — keeps the cached system block effective).
- Hand-rolled `Orchestrator`: pre-intent retrieval, post-completion recording, retrieval-driven hint emission.
- LangGraph `bootstrap_turn`: retrieval. `extract_intent`: passes block. `evaluate_hint`: retrieval-driven hint takes precedence over the heuristic. `complete_turn`: recorder fires.
- `engine.ts`: factory wires the memory subsystem into both engines.
- Both engines remain identical in event-stream output (parity test still green).

### Memory hint upgrade
- The existing `concierge.memory.hint` event now fires from cross-session retrieval when a memory matches above the floor. The Slice A9 in-session heuristic remains as the fallback when retrieval returns nothing — same single-hint-per-turn UI surface, richer underlying signal.

### Env + features
- `getServerFeatures().memory` surfaces `{ kind: 'in-memory' | 'pgvector', embedding: 'bag-of-words' | 'anthropic' }` for the `/admin` dashboard.
- `.env.example` documents `STAYSCOUT_PGVECTOR` and `STAYSCOUT_USE_ANTHROPIC_EMBEDDINGS`.

### Tests (+26)
- `tests/bag-of-words-embedding.test.ts` (+8): determinism, dimensionality, L2 unit norm, semantic ordering, similarity bounds.
- `tests/memory-store.test.ts` (+8): round-trip, owner isolation, top-K, score floor, FIFO eviction, clearOwner.
- `tests/memory-recorder.test.ts` (+5): records expected memory kinds, structural snapshot content, idempotency, failure isolation.
- `tests/memory-retriever.test.ts` (+5): empty store, score floor null, prompt formatting, top-K cap, owner isolation.

## Architectural decisions (recorded)

**1. Memory is owned, not global** — same key as trips: `{ownerKind, ownerId}`.

**2. Retrieval enriches, never replaces, intent inference** — model still does extraction; memories appear as a `<memory>` block in the user prompt, marked as soft hints.

**3. The InMemory implementation is real, not stub** — `pnpm dev` produces visible memory behavior on turn 2 without DB or embeddings.

**4. Embeddings are pluggable + explicit** — Anthropic embeddings opt-in via `STAYSCOUT_USE_ANTHROPIC_EMBEDDINGS=1`. Bag-of-words is the default even with an Anthropic key.

**5. Recording is post-completion, idempotent, bounded** — fires after `turn.completed`; ≤3 records per turn; idempotent on `turnId`; failures never block the user.

**6. Retrieval is conservative** — top-K=3, score floor 0.20 (BOW). Below the floor, skip — junk memories don't pollute the prompt.

**7. UI surface unchanged** — `concierge.memory.hint` event already existed; C1 just populates it from real recall.
