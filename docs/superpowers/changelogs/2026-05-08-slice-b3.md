# Slice B3 - Saved Trips Resurfacing + Share Links

**Tag:** `slice-b3`
**Plan:** [`docs/superpowers/plans/2026-05-08-slice-b3-resurfacing-and-share-links.md`](../plans/2026-05-08-slice-b3-resurfacing-and-share-links.md)

## Goal

Saved trips become first-class - clicking one resurfaces the proposal on the canvas; sharing one produces a public read-only URL anyone can open, with a "Save to my StayScout" CTA so recipients can clone it into their own bucket.

## What shipped

### Schema + slug
- `Trip.shareSlug String? @unique` - lazy-minted on first share
- `src/lib/session/share-slug.ts` - 16-char base62 (~95 bits entropy), URL-safe, generated via `crypto.getRandomValues` with rejection sampling for uniformity
- `isValidShareSlug()` for cheap pre-DB validation

### SessionStore extensions
- `mintShareSlug({ownerKind, ownerId, tripId})` - owner-gated, idempotent. Returns existing slug if one exists, else mints + persists. Returns `null` for unowned/missing trips (route handler maps to 404).
- `getTripBySlug(slug)` - public read, returns sanitized `SharedTrip` (no `ownerId`, no `conversationId`, `intent.rawInput` masked). Implementation lives at the SessionStore boundary so privacy is invariant.
- Both `InMemorySessionStore` and `PostgresSessionStore` implement. In-memory uses a side index `tripsBySlug: Map<slug, SavedTrip>` for O(1) lookup; Postgres relies on the `@unique` index. Migration carries the slug forward (still resolvable post anon→user move).

### API routes
- `POST /api/trips/[tripId]/share` - owner-gated, mints slug. Returns `{slug, url}` where `url = ${origin}/t/${slug}`.
- `GET /api/trips/by-slug/[slug]` - public, no auth. Returns sanitized `SharedTrip` or 404. Caches `public, max-age=60, stale-while-revalidate=300` - slugs are immutable, owner deletes propagate within 60s.

### Public `/t/[slug]` page
- Server component. Fetches via `getSessionStore().getTripBySlug(slug)`. 404s on invalid/unknown slug.
- `SharedTripView` renders read-only - hero stay + alternatives + reasoning highlights, no canvas chrome
- Open Graph metadata via `generateMetadata` so links unfurl with destination + hero name + reasoning summary
- "Save to my StayScout" CTA → POST /api/trips/save with the proposal+intent+proposalRef. Idempotent per-recipient (each recipient owns their own row).
- "Try this prompt myself" link → `/?prompt=<urlencoded>`. Falls back to a structured-intent reconstruction when `rawInput` is masked.

### Workspace integration
- `useWorkspaceStore.resurfaceSavedTrip()` - pushes a synthetic settled `Turn` onto `turns`, sets `currentTurnId`, closes the saved panel. De-dupe on `turnId`. Canvas renders it like any other turn.
- `SavedTripRow` - clicking the body resurfaces; share + remove buttons exposed on hover
- `ShareModal` - slug-mint on open (lazy), Web Share API on mobile (single-tap to native share sheet), clipboard fallback
- `useSavedTrips.share(tripId)` - caches the slug on the local row after mint so re-shares don't re-POST
- `UrlInit` - consumes `?prompt=<text>` (seeds `inputDraft`) and `#saved` (opens panel), strips both via `router.replace('/')` so back-nav doesn't re-apply

## Architectural decisions (recorded for B4+)

**1. Share slugs are unguessable, not authenticated.** 16-char base62 ≈ 95 bits - no expiry needed in B3.

**2. Slug is on Trip, lazy-minted.** Most trips never get shared; saving never mints. `@unique` constraint enforced at DB level.

**3. Public read returns sanitized projection.** `SharedTrip` strips owner-identifying fields and `intent.rawInput` (which can echo names/locations). Sanitization lives at the SessionStore boundary, not the route handler - privacy is invariant.

**4. Resurfacing is a synthetic turn.** Pin/compare/detail UIs already work against a `Turn` record - no ripple changes downstream.

**5. Save-from-share is a fork.** Recipients get their own row; original owner's trip is untouched.

**6. Refining a resurfaced trip is out of B3 scope.** Cross-process priorTurn lookup may 404. Documented as B3.x follow-up; the workspace gracefully degrades - refine still works for trips originally created in this Node process.

## Mock-safe matrix (unchanged from B2)

The new share/resurface paths run in either mode:
- In-memory: shares survive within a dev session
- Postgres: shares are durable, cross-process, cross-device
